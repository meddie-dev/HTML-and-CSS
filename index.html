ANIMATION IN CSS

Animation properties

animation-name
- name of the @keyframe variable. specifies the names of one or more @keyframes at-rules that describe the animation to apply to an element.
animation-duration
- the duration of the animation in @keyframe variable.
animation-direction
- sets whether an animation should play forward, backward, or alternate back and forth between playing the sequence forward and backward.
  • normal
    • - Default value. The animation is played as normal (forwards).
  • reverse
    • - The animation is played in reverse direction (backwards).
  • alternate
    • - The animation is played forwards first, then backwards.
  • alternate-reverse
    • - The animation is played backwards first, then forwards.
animation-play-state
- property specifies whether the animation is running or paused.
  • paused
    • - Specifies that the animation is paused.
  • running
    • - Default value. Specifies that the animation is running.
animation-timing-function
- specifies the speed curve of an animation.
  • linear
    • - The animation has the same speed from start to end.
  • ease
    • - Default value. The animation has a slow start, then fast, before it ends slowly.
  • ease-in
    • - The animation has a slow start.
  • ease-out
    • - The animation has a slow end.
  • ease-in-out
    • - The animation has both a slow start and a slow end.
  • step-start
    • - Equivalent to steps(1, start).
  • step-end
    • - Equivalent to steps(1, end).
  • steps(int,start|end)
    • - Specifies a stepping function, with two parameters.
  • cubic-bezier(n,n,n,n)
    • - Define your own values in the cubic-bezier function Possible values are numeric values from 0 to 1.
animation-iteration-count
- specifies the number of times an animation should be played.
  • number
    • A number that defines how many times an animation should be played. Default value is 1
  • infinite
    • Specifies that the animation should be played infinite times (for ever)
@keyframe
- This gives more control over the intermediate steps of the animation sequence than transitions.
